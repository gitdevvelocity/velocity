## 🔧 Common Issues & Fixes

**❌ Issue: "Injection Failed" or Velocity freezes on inject**
> **Fix:** Run `VelocityExecutor.exe` as Administrator. Make sure Roblox is **fully loaded into a game session** before clicking Inject — injecting on the loading screen causes failures. Temporarily disable Windows Defender and retry.

**❌ Issue: Antivirus quarantines or deletes the Velocity EXE**
> **Fix:** This is a documented false positive. All DLL injectors trigger AV software by behavioral signatures, not actual malicious code. Add the entire Velocity folder to your antivirus exclusion list, re-extract the ZIP, and try again. Users asking **"is velocity executor a virus"** can confirm safety by uploading the file to VirusTotal — it consistently returns clean results from major engines.

**❌ Issue: Script executes but nothing happens in-game**
> **Fix:** Confirm injection was successful (green indicator). Try a different, known-working script from the built-in Script Hub to rule out script-specific issues. If the game was updated recently, some scripts may need to be updated by their authors.

**❌ Issue: Auto-Execute not firing on game join**
> **Fix:** Go to Settings → Auto-Execute and verify your script is checked in the queue. Ensure the "Enable Auto-Execute" master toggle is ON. Scripts in the queue only fire after a successful injection — confirm inject completes before the script is expected to run.

**❌ Issue: "Missing .NET Framework" error on startup**
> **Fix:** Download and install **.NET Framework 4.8** from Microsoft's official download page. This is a required dependency for the Velocity launcher.

**❌ Issue: Multi-instance injection crashes on the second window**
> **Fix:** Ensure each Roblox instance is fully loaded before injecting into the second one. Injecting into multiple windows simultaneously (rather than sequentially) is a known instability trigger. Inject one window, confirm success, then inject the next.

**❌ Issue: Velocity UI doesn't appear after successful injection**
> **Fix:** Press **Right Shift** or check Settings for the assigned UI toggle keybind. Some script hubs override the default keybind — re-check after running any hub script.
